# 内容自动化

> 撰写 / 标题 / 排版

通过 OpenClaw Agent + Skill，实现内容撰写、标题优化、排版规范化等自动化。

---

## 一、内容撰写

### 使用方式

在聊天中描述需求，例如：

- 「写一篇 800 字关于某某主题的文章，结构：引言-三个论点-结论」
- 「根据下面的大纲扩展成完整文章」
- 「把这段口语化内容改成正式书面语」

Agent 根据描述生成初稿，可多轮修改。

### Skill 注入风格

```markdown
# ~/.openclaw/skills/content-write/SKILL.md

# 内容撰写技能

## 指令
撰写规范：
- 结构清晰：小标题层级分明
- 语言：根据受众选择口语/书面
- 长度：严格按用户指定的字数控制
- 避免空洞套话，多用具体案例

若用户提供大纲，按大纲逐段扩展；若未提供，先输出大纲供确认再撰写。
```

## 二、标题优化

### 场景

- 为已有文章生成 3–5 个备选标题
- 优化现有标题，提升点击率
- 生成符合平台规范的标题（如字数限制）

### 示例指令

「为下面这篇文章生成 5 个吸引点击的标题，风格偏悬念/干货/情感，每个不超过 20 字」

---

## 三、排版规范化

### 常见需求

- 统一标题层级（# ## ###）
- 中英文、数字与中文间加空格
- 列表格式统一（- 或 1. ）
- 段落长度控制，避免过长
- 添加目录（TOC）

### 斜杠命令

```json5
{
  slashCommands: {
    "/排版": {
      description: "规范化排版",
      prompt: "对上述内容进行 Markdown 排版规范化：统一标题层级、中英文空格、列表格式、段落拆分，并生成目录。"
    }
  }
}
```

## 四、多平台适配

不同平台有不同规范（如公众号、知乎、小红书），可通过 Skill 区分：

```markdown
# ~/.openclaw/skills/platform-format/SKILL.md

# 平台格式技能

## 指令
- 公众号：段落短、多 emoji、可加引导关注
- 知乎：偏干货、可长文、引用规范
- 小红书：短句、多换行、话题标签格式 #xxx
- 通用：Markdown 标准格式
```

使用时可说：「按小红书格式改写上面这段」。

## 五、批量生成（Cron）

```json5
{
  cron: {
    jobs: [
      {
        schedule: "0 9 * * 1-5",
        agent: "content-daily",
        message: "根据今日素材主题生成一篇 500 字短文，推送到 session content-draft"
      }
    ]
  }
}
```

---

## 六、注意事项

- 生成内容需人工审核后再发布
- 注意版权，避免大段复制他人文章
- 不同平台有不同合规要求，需单独确认
