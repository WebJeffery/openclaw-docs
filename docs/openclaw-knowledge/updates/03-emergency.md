# 紧急更新

> 重大版本 / 高频坑，第一时间补充

紧急更新的触发条件、推送方式与历史记录。

---

## 一、触发条件

| 类型 | 说明 |
|------|------|
| **重大版本** | OpenClaw 大版本升级，存在 Breaking Changes |
| **高频坑** | 大量用户反馈同一问题，需快速补充避坑文档 |
| **安全提示** | 发现安全风险或配置不当导致的泄露等 |
| **官方重要变更** | API、配置结构、命令等重大调整 |

---

## 二、推送方式

- **知识库**：立即更新对应章节，更新日志标注「紧急」
- **社群**：群内 @all 或置顶通知
- **邮件/站内信**：若已绑定，同步推送（视平台能力）

---

## 三、紧急更新记录

（示例格式，以实际为准）

| 日期 | 主题 | 说明 |
|------|------|------|
| 待补充 | OpenClaw v2026.x 迁移 | Breaking Changes 与迁移步骤 |
| 待补充 | 某通道 Token 过期处理 | 高频问题补充 |

完整历史见 [版本日志](/openclaw-knowledge/appendix/03-changelog)。

---

## 四、与常规/季度的区别

| 维度 | 紧急更新 | 常规/季度 |
|------|----------|------------|
| **时效** | 发现即发，通常 24–48 小时内 | 按计划排期 |
| **优先级** | 最高 | 正常 |
| **形式** | 单篇补充或修订 | 按计划批量 |

---

## 五、如何反馈紧急问题

- 通过 [一对一答疑](/openclaw-knowledge/module-05-benefits/01-support) 反馈
- 在交流群内描述问题与复现步骤
- 若为普遍性问题，会纳入紧急更新排期
