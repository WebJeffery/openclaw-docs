# 合规红线

> 规避法律 / 技术风险

部署与使用 OpenClaw 时，需注意数据合规、内容合规与平台规则，避免法律与技术风险。

---

## 一、数据合规

### 数据出境

- 使用境外 API（Anthropic、OpenAI 等）时，对话内容会发送到境外服务器
- 涉及个人信息、敏感数据的场景，需评估是否符合《个人信息保护法》等要求
- 可选：使用境内 Provider（通义、文心等）或 Ollama 本地部署

### 数据留存

- 会话、配置默认存储在 `~/.openclaw`
- 需定期清理或加密存储时，可配置独立 state 目录并配合备份策略
- 离职、设备交接时，注意清除本地凭证与会话数据

---

## 二、内容合规

### 生成内容

- Agent 生成的内容可能含违规、敏感信息，需人工审核后再对外发布
- 可配置 Skill 注入合规指令（如「不生成违法、侵权内容」）
- 自媒体、金融等场景需额外遵守行业规范

### 输入内容

- 避免向 Agent 输入明显违法、侵权、涉密内容
- 若接入公开群组，建议配置 `requireMention`、白名单等，控制触发范围

---

## 三、平台规则

### 通道服务条款

- Telegram、Discord、飞书等各有服务条款与机器人规范
- 自动化消息需注意频率、内容，避免被判定为 spam 或滥用
- 部分平台禁止或限制自动化回复，需提前确认

### API 使用条款

- 各 AI Provider（Anthropic、OpenAI 等）有 API 使用条款
- 禁止将输出用于训练竞争模型、大规模爬取等
- 遵守各家的内容政策与地域限制

---

## 四、金融与投研

- A股、基金等场景的数据使用需符合数据源协议
- 生成的分析、建议不得作为投资建议对外提供
- 注意与「荐股」「代客理财」等行为的边界

---

## 五、企业内部

- 若部署在企业环境，需符合内部信息安全与合规要求
- 建议与法务、信息安全团队沟通部署范围与数据流向
- 可配置沙箱、白名单、审计日志等满足内控要求

---

## 六、建议措施

1. **最小权限**：按需配置通道、Agent、工具权限
2. **审计**：定期检查日志、会话、配置变更
3. **培训**：对使用人员做合规与安全培训
4. **预案**：制定数据泄露、违规内容等应急处理流程
