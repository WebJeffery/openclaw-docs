# 违规词检查 + 重复内容优化

> 合规与去重

通过 OpenClaw Agent + Skill，实现违规词检查与重复内容优化，降低内容风险。

---

## 一、违规词检查

### 思路

1. 维护违规词库（敏感词、平台禁词、广告法禁用词等）
2. Agent 读取词库，对用户提供的内容进行扫描
3. 标注命中位置，并给出替换建议

### Skill 示例

```markdown
# ~/.openclaw/skills/compliance-check/SKILL.md

# 违规词检查技能

## 描述
对文本进行违规词、敏感词检查，标注位置并建议替换。

## 使用方法
当用户请求「检查违规词」「敏感词检测」时启用。

## 指令
检查流程：
1. 加载违规词库（可从用户提供的文件或预设列表）
2. 对目标文本逐词/逐句扫描
3. 命中处标注：原词、位置、风险等级、建议替换
4. 输出格式：
   ## 违规词检查结果
   - [高] 第3段「xxx」→ 建议替换为「yyy」
   - [中] 第5段「xxx」→ 需人工确认
5. 若无命中，输出「未发现违规词」
```

### 词库维护

可将词库放在 `~/.openclaw/skills/compliance-check/words.txt`，Skill 中说明路径，Agent 通过 `read_file` 读取。

---

## 二、重复内容优化

### 场景

- 多篇素材整合时去重
- 与已有发布内容对比，避免重复
- 同义改写，降低重复率

### 使用方式

「下面两段内容有重复，请合并并去重，保留核心信息」

「把这段改写成意思相同但表述不同的版本，降低与原文的重复率」

### Skill 注入

```markdown
# ~/.openclaw/skills/dedup-rewrite/SKILL.md

# 去重与改写技能

## 指令
- 合并多段内容时：识别重复句/段，保留最完整表述，删除冗余
- 同义改写时：保持原意，更换句式、用词、语序，避免连续 5 字以上与原文相同
- 输出时标注「已去重」或「已改写」的段落
```

## 三、斜杠命令

```json5
{
  slashCommands: {
    "/合规": {
      description: "违规词检查",
      prompt: "对上述内容进行违规词检查，标注命中词及建议替换。"
    },
    "/去重": {
      description: "内容去重",
      prompt: "对上述多段内容合并去重，保留核心信息，删除重复表述。"
    },
    "/改写": {
      description: "同义改写",
      prompt: "将上述内容改写成意思相同但表述不同的版本，降低重复率。"
    }
  }
}
```

---

## 四、平台差异

不同平台违规词库不同（如公众号、抖音、小红书），可：

- 为不同平台建不同 Skill 或词库
- 在指令中指定平台，Agent 按对应规则检查

---

## 五、注意事项

- 词库需定期更新，关注平台规则变化
- 机器检查有漏检可能，重要内容建议人工复核
- 改写需注意不改变原意，避免曲解
