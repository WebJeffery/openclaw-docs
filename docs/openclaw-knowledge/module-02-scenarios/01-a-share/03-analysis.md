# 个股深度分析

> 自动生成评估报告

通过专用 Agent + 技能 + 数据脚本，实现个股深度分析报告的自动生成。

---

## 一、分析框架设计

### 报告结构示例

1. **基本信息**：代码、名称、所属板块、市值
2. **行情概览**：近期涨跌、成交量、技术形态
3. **财务摘要**：营收、利润、ROE、估值
4. **资金流向**：主力资金、北向资金
5. **风险提示**：估值、波动、行业风险
6. **综合结论**：简要评级与关注点

---

## 二、专用分析 Agent

```json5
{
  agents: {
    "stock-analyst": {
      model: { primary: "anthropic/claude-3-5-sonnet" },
      systemPrompt: {
        content: |
          你是 A股个股分析助手。当用户提供股票代码或名称时：
          1. 通过可用工具获取基本面、行情、资金流向等数据
          2. 按「基本信息 → 行情 → 财务 → 资金 → 风险 → 结论」结构组织
          3. 输出 Markdown 格式报告，数据需标注来源与时间
          4. 结论部分需明确区分「事实」与「推测」，不做投资建议
      }
    }
  }
}
```

## 三、分析 Skill 示例

```markdown
# ~/.openclaw/skills/stock-analysis/SKILL.md

# 个股深度分析技能

## 描述
对单只 A股进行多维度分析，生成结构化评估报告。

## 使用方法
当用户请求「分析某某股票」「某某股票怎么样」时启用。

## 指令
分析流程：
1. 确认股票代码（支持 000001.SZ、600000.SH 等格式）
2. 调用数据脚本获取：近期行情、财务指标、资金流向
3. 若数据不足，可用 web_search 补充公开资讯
4. 按模板输出报告，包含：
   - 数据来源与更新时间
   - 估值水平（PE、PB）与历史分位
   - 主要风险点
   - 结论（不超过 3 条）
```

---

## 四、斜杠命令快捷触发

```json5
{
  slashCommands: {
    "/分析": {
      description: "个股深度分析",
      prompt: "请对以下股票进行深度分析，生成完整评估报告：",
      agent: "stock-analyst"
    }
  }
}
```

使用方式：在聊天中发送 `/分析 000001.SZ` 或 `/分析 平安银行`。

---

## 五、定时生成报告（Cron）

```json5
{
  cron: {
    jobs: [
      {
        schedule: "0 18 * * 1-5",
        agent: "stock-analyst",
        message: "对持仓列表 [000001.SZ, 600036.SH] 生成今日分析简报，推送到 session market-reports"
      }
    ]
  }
}
```

---

## 六、数据与合规

- 财务、行情数据需通过合规 API 获取
- 报告需注明「仅供参考，不构成投资建议」
- 避免使用未授权的付费研报原文
