# 2026 最新版升级亮点

> A股场景优化、手机端支持等

OpenClaw 2026 系列（如 v2026.3.7）在架构、多端支持、配置和工具方面有大量更新。本文汇总与日常使用最相关的亮点。

---

## 多端与移动端支持

### iOS / 手机端

- **iOS App**：支持在 iPhone / iPad 上使用 OpenClaw，Bundle ID 统一为 `ai.openclaw.client`
- **Watch 应用**：图标与元数据更新，支持 Apple Watch 端
- **App Store Connect**：支持 Fastlane 元数据与截图自动化，便于上架

### 多终端适配

- 手机端可与 Gateway 配对，在移动设备上收发消息
- 支持远程 Gateway（通过 Tailscale 等内网穿透访问家中/公司部署）

---

## 通道与路由增强

### Telegram

- **Topic 级 Agent 路由**：论坛群组、话题可绑定独立 Agent，会话隔离
- **ACP Topic 绑定**：`/acp spawn` 支持 `--thread here|auto`，话题内跟进自动路由到 ACP 会话
- **审批按钮**：Telegram 内可直接点击审批，无需切到终端

### Discord

- **allowBots: "mentions"**：仅在被 @ 时响应 Bot 消息，减少误触发
- **持久化 Channel 绑定**：ACP 线程目标持久化存储，重启后仍生效

### Slack

- **DM 打字反馈**：`channels.slack.typingReaction` 可在 Socket Mode DM 中显示处理状态

---

## 智能体与上下文

### 上下文与压缩

- **ContextEngine 插件**：支持插件实现自定义上下文管理（如 lossless-claw）
- **压缩后段落可配置**：`agents.defaults.compaction.postCompactionSections` 控制压缩后重新注入的 AGENTS.md 段落
- **压缩生命周期钩子**：`session:compact:before`、`session:compact:after`，便于自动化响应

### 工具与结果

- **工具结果截断**：超大结果采用 head+tail 截断，保留关键诊断信息
- **Diffs 引导**：从无条件注入改为通过插件技能注入，减少无关提示词噪音

---

## 配置与安全

### Gateway 鉴权

- **BREAKING**：当同时配置 `gateway.auth.token` 和 `gateway.auth.password` 时，必须显式设置 `gateway.auth.mode` 为 `token` 或 `password`
- **SecretRef**：`gateway.auth.token` 支持 SecretRef，便于密钥管理

### 插件与钩子

- **allowPromptInjection**：插件钩子可配置是否允许修改提示词，提升安全性
- **prependSystemContext / appendSystemContext**：静态引导可放入系统上下文，利于 Provider 缓存、降低 Token 成本

---

## 部署与运维

### Docker

- **多阶段构建**：Dockerfile 重构为多阶段构建，运行时镜像更小，无构建工具和源码
- **OPENCLAW_VARIANT=slim**：支持 bookworm-slim 变体
- **OPENCLAW_EXTENSIONS**：容器构建时可预装扩展依赖，加快启动

### 其他

- **Cron 任务快照**：`jobs.json.bak` 在规范化持久化时不再覆盖，便于恢复
- **无头 Linux 探针**：`systemctl --user is-enabled` 失败时不再导致安装流程崩溃

---

## 工具与搜索

- **Perplexity**：切换为 Search API，支持结构化结果及语言/地区/时间过滤
- **Brave 搜索**：语言码校验与别名映射（如 zh、ja → zh-hans 等）
- **xAI/Grok**：避免与原生 web_search 工具冲突
- **Onboarding**：向导中增加 Web 搜索提供商选择与完整列表

---

## 升级建议

1. **鉴权配置**：若同时使用 token 和 password，务必设置 `gateway.auth.mode`
2. **Docker 用户**：可尝试 `OPENCLAW_VARIANT=slim` 减小镜像体积
3. **Telegram 高级用法**：利用 Topic 路由和 ACP 绑定做多场景隔离
