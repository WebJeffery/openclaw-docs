# 界面导航

> 核心功能快速熟悉

OpenClaw 的主要交互发生在 **聊天软件内**（Telegram、飞书等），另有 **Web 控制台** 和 **命令行** 用于配置与运维。

---

## 一、主入口：聊天软件

日常使用 AI 的界面就是你已熟悉的聊天 App：

- **Telegram**：与 Bot 私聊或群聊
- **飞书**：在群组或私聊中 @ Bot
- **Discord**：在频道或 DM 中发消息

**操作方式**：像和真人聊天一样发消息，AI 会回复。首次使用需完成配对（见 [通讯渠道配对](/openclaw-knowledge/module-01-basics/03-basics/03-channels)）。

---

## 二、Web 控制台（Dashboard）

用于配置、配对、查看状态，不是主聊天界面。

### 打开方式

```bash
openclaw dashboard
```

浏览器会自动打开控制台地址（通常为 `http://localhost:18789` 或配置的 URL）。

### 主要功能

| 功能 | 说明 |
|------|------|
| **配对管理** | 查看、批准、拒绝待审批的配对请求 |
| **通道状态** | 各通道连接状态、错误信息 |
| **配置编辑** | 修改模型、技能、通道等配置 |
| **日志查看** | 实时日志流 |

---

## 三、命令行入口

### 常用命令速查

| 命令 | 作用 |
|------|------|
| `openclaw gateway status` | 查看网关状态 |
| `openclaw gateway start/stop/restart` | 启停网关 |
| `openclaw channels status` | 通道状态 |
| `openclaw pairing list <channel>` | 待审批配对 |
| `openclaw pairing approve <channel> <code>` | 批准配对 |
| `openclaw doctor` | 一键诊断 |
| `openclaw logs --follow` | 实时日志 |
| `openclaw configure` | 配置向导 |
| `openclaw dashboard` | 打开 Web 控制台 |

### 配置相关

```bash
openclaw configure                 # 完整配置向导
openclaw configure --section model # 只改模型
openclaw onboard                   # 首次安装向导
```

---

## 四、聊天内的斜杠命令

在 Telegram、Discord 等聊天窗口输入 `/` 可看到可用命令，例如：

| 命令 | 说明 |
|------|------|
| `/think` | 深度思考模式 |
| `/verbose` | 显示推理过程 |
| `/reset` | 清除会话，重新开始 |
| `/model` | 切换 AI 模型 |
| `/help` | 显示帮助 |
| `/stop` | 停止当前操作 |

详见 [核心指令入门](/openclaw-knowledge/module-01-basics/03-basics/02-commands)。

---

## 五、推荐使用路径

1. **日常对话**：在 Telegram/飞书 里直接发消息
2. **配置与配对**：用 `openclaw dashboard` 或 `openclaw configure`
3. **排查问题**：用 `openclaw doctor`、`openclaw logs --follow`
4. **高级操作**：用斜杠命令或自定义命令
